In this section, you will run an example use case application, the KnowCOVID-19 web application, which is a recommender system for clinical researchers, healthcare professionals to automatically filter high-quality publications according to the evidence-based information filtering standard.

4.1 KnowCOVID-19 Application Background.

KnowCOVID-19 web application is an “evidence-based” recommender system that utilizes edge computing service to integrate recommender modules for data analytics using end-user thin-clients. The edge computing service features chatbot-based web interface that handles a given COVID-19 publication dataset using two recommender system modules: (i) evidence-based filtering that observes domain specific topics across the literature and classifies the filtered information according to a clinical category, and (ii) social filtering that allows diverse experts with similar objectives to collaborate via a “social plane” to jointly find answers to critical clinical questions to fight the pandemic. KnowCOVID-19 design of using a novel AI-based literature review service aims to reduce the manual burden in traditional literature review workflows of data consumers who laboriously search and filter continuously-growing publication databases for high quality information.

The main recommender module features evidence-based filtering, which observes latent knowledge patterns across the literature documents using a Domain-Specific Topic Model (DSTM) that generates a Dirichlet probability distribution set of keywords in each topic. Figure4.1illustrates the Levels of the Evidence Pyramid showing filtered and unfiltered information to help visualize both the quality of evidence and the amount of evidence available. The levels of evidence pyramid provides a way to visualize both the quality of evidence and the amount of evidence available. For example, systematic reviews are at the top of the pyramid, meaning they are both the highest level of evidence and the least common. As you go down the pyramid, the amount of evidence will increase as the quality of the evidence decreases. For more information regarding to this project, please reference a published paper which is accessible through the link:KnowCOVID-19 Paper.

Figure 4.1: Levels of the Evidence Pyramid.

We have implemented the KnowCOIVD-19 web application that includes three different modules, i.e., Java-based front end web portal for users to interact with the application, a MySQL database to hold the user data as well as the query data, and a Python microservice server that executes the user’s queries by running machine learning algorithms and recommender models. These modules will be deployed into Docker containers, and integrated together via Docker compose orchestration. For the purpose of this lab practice, we have prepared the scripts to create Docker containers for the modules, and all you have to do is follow the lab steps below to deploy them all on the same VM instance (i.e. on the Jenkins server), and create a Jenkins pipeline to automatically deploy the application when your code is updated through GitHub repository.

4.2 Clone our KnowCOVID-19 GitHub repository

You will need to clone our prepared KnowCOVID-19 GitHub repository for this lab. The GitHub repo is accessible here. You will need to fork the repository into your own GitHub account to be able to make updates to the code and run the pipeline in your Jenkins server.

4.3 Create KnowCOVID-19 pipeline in Jenkins.

From your Jenkins server main page, create a new pipeline by clicking “New Item”, give it a name like “knowcovid-19”, and select “Pipeline”. Click OK to create the pipeline.

In the pipeline configuration, go to “Build Triggers”, and check “GitHub hook trigger for GITScm polling”. This will allow the Jenkins pipeline to find if there is a change in the GitHub repo. If there is a change, it will automatically initiate a build of the pipeline.

Scroll down in the configuration page. In the “Pipeline” section, click the drop down menu and select “Pipeline script from SCM”. This setting will enable the Jenkins pipeline to find the Jenkinsfile in the GitHub repository and execute the build based on the Jenkinsfile. Then in the “SCM”, input your forked KnowCovid-demo repository (which will have your OWN GitHub ID!). Leave “Credentials” as none, and Branch as “*/master”.

In “Script Path”, specify the location of the Jenkinsfile, and input “Jenkinsfile”, which means the Jenkinsfile at the root directory of the GitHub repository. Then click “Save”.

Go to your forked GitHub repository, click “Settings”, then “Webhooks”, and then “Add webhook”. In. the webhook form, add “http://YourServerIP:8080/github-webhook/” by replacing the “YourServerIP” with your actual IP address, and select “application/json” in “Content type”. Leave others as default, and click “Add webhook”.

4.4 Build your pipeline

At this point, you are ready to build your pipeline for the first time.

4.4.1 Check the Jenkinsfile code.

But before you run it, lets take a look at the Jenkinsfile and see how it is done. It actually looks very simple because we are using Docker compose to create Docker container orchestration under the hood, so Jenkins has only a little to worry about, i.e., initiating Docker compose in the build process:

pipeline {
    agent any 
    stages {
        stage('Build') {
            steps {
                sh 'docker-compose up -d --build'
                }
                post {
                    success {
                        echo 'Build successful. Now archiving...'

                    }
            }
        }
    }
}

4.4.2 Modify the GitHub repository.

Go to your forked GitHub repository, and modify this file:

KnowCovid-demo/covid-app/Covid19-website/src/main/resources/templates/index.html

by changing the following line:

<!-- <h1 style="text-align: center;"> This is Version 1! </h1> -->

to (basically you are just uncommentting and activating the code!)

<h1 style="text-align: center;"> This is Version 1! </h1>

Then you will need to modify this file: KnowCovid-demo/covid-app/Covid19-website/src/main/resources/static/js/angular_services/mainService.js by updating the following (line 4):

var serverURL = "http://****:9090/Covid-19/";

to

var serverURL = "http://Your_Server_IP:9090/Covid-19/";

4.4.3 Turn off the Tomcat server on your VM instance

In our earlier sections, you have used Tomcat server that was hosted on the Jenkins server. Since you are running KnowCovid-19 with Tomcat in a Docker container, you will need to turn off the Tomcat on the host machine to avoid address conflicts. From your EC2 instance SSH terminal, run

$ sudo /usr/local/tomcat/bin/shutdown.sh

4.4.4 Change permissions on Docker folder

Docker may fail to run because of permission limitations to pull the user information for starting the compose command. To change this, run the following command on your EC2 instance SSH terminal:

$ sudo chmod 666 /var/run/docker.sock

4.4.5 Build the pipeline.

Go back to your Jenkins portal, click the KnowCovid-19 pipeline, and click “Build Now”. You will see that the pipeline start to build. In a short while, ranging from a few seconds to a couple of minutes (depending on if you are running the pipeline for the first time), you will see that the build has completed successfully.

Warning: If you are seeing that the build has failed by showing you a red “1”, click the “1” and then “Console output”. You will see the logs of the build process. From there you can see what has went wrong. Mostly the issue will come from that the jenkins user does not have the access to Docker Daemon. For this issue, you can do the following: - SSH login to your EC2 instance - Run the following commands: 
$ usermod -aG docker jenkins
Now build the pipeline again. It should work.


Once the build has successfully completed, open up your web browser, and connect to “http://YourServerIP:9090/Covid- 19”. You will see that the KnowCovid-19 web portal is up and running, as shown below. Notice that at the bottom of the home page, there is a line of text showing “THIS IS VERSION 1 !”, which indicates that you have successfully uncommentted this text line in the “index.html” file in your forked repository.

4.5 Use the KnowCovid-19 application

To use the KnowCovid-19 web application, you will first need to create an account and sign in it. Click “Login” from the top right corner, and then “Go and Register” if you do not have an account. Once you have registered, you will be redirected to the login page. Use your registered information to login.

After logging in, you can read through some information in the “About”, “Data” and “Reports” pages. Now click the “Search” page. This page contains two functions: topic filter and search function. The topic filter uses evidence-based filtering, which observes latent knowledge patterns across the literature documents and generates a Dirichlet probability distribution set of keywords in each topic. The search engine uses a Python library for topic modelling, document indexing and similarity retrieval with large corpora, using popular algorithms, such as online Latent Se-mantic Analysis (LSA/LSI/SVD), Latent Dirichlet Allocation (LDA), Random Projections (RP), Hierarchical Dirichlet Process (HDP) or word2vec deep learning.

Feel free to run some filter/search queries using these two engines, and see if they can give you the results that you are expecting to see!

4.6 Update the web application.

Up to this point, you are running the Version 1 of the KnowCovid-19 web application. Now try to modify the code base, and see if Jenkins can automatically integrate the changes and deploy a newer version of the application!

4.6.1 Update GitHub repository

From your forked GitHub repository, change the following code with a simple update. Go to your forked GitHub repository, and modify this file:

KnowCovid-demo/covid-app/Covid19-website/src/main/resources/templates/index.html

by changing the following line:

<h1 style="text-align: center;"> This is Version 1! </h1>

to

<h1 style="text-align: center;"> This is Version 2! </h1>

Save the change in your repository.

4.6.2 Check for new build

Go back to Jenkins pipeline page. You will immediately notice that there is a new build initiated on your pipeline.

After a few seconds, the build will be completed, and you will see the pipeline in the stage view as shown below, with some information on the building time, history, and etc.

Go back to your web browser running your KnowCovid-19 portal, refresh the page. It will take a few seconds for the page to re-appear. Once you see the home page appears again, you will see that the version message has been changed to “THIS IS VERSION 2 !”. This means that Jenkins has rebuilt and re-deployed your KnowCovid-19 web portal by integrating the changes you made to your GitHub repository, i.e. it has completed a cycle in the CI/CD process.

Note: Keep in mind that, we are running the three application modules in Docker containers locally inside an AWS EC2 instance. So your Jenkins pipeline does not automatically update MySQL database deployed in the Docker container. That means if you rebuild your pipeline and re-deploy your web application, the data you generated in the last build could be gone, unless you have made a backup of the database!
