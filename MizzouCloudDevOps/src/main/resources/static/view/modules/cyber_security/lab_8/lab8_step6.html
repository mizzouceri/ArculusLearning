In this chapter, we will deploy Video Straming Application in a separate AWS EC2 instance and export the application metrics through a custom prometheus exporter. The stack contains a video streaming server which will be hosting the video for client and the video steaming client which requests the video from server and replicates the video stream on AWS. Video streaming server logs the metrics into a log file. Few of the metrics logged by the server are number of http bypes transferred(fabric_http_bytes_sent), stream duration(fabric_stream_duration) and throughput(fabric_throughput). The prometheus exporter deployed in the node analyzes the log file and exposes the metrics to prometheus.

Prometheus server deployed in the first EC2 instance scrapes the metrics exposed by the custom prometheus exporter deployed in the second EC2 instance. You can access the prometheus portal and see the metrics by executing them in query console. You can visualize the same metrics on Grafana.
