4.5 Publish the application

By Default all Application Pods that get created do not allow external traffic or traffic from other Application Pods. In order to receive traffic we need to publish the app to have a service. Since we are running the cluster on AWS platform, we have the option to use a Load Balancer provided by AWS. Let's create a Load Balancer service for the Pet Clinic application. This will allow external traffic to reach the application inside our cluster.

  • Publish Pet Clinic using a service named "petclinic-tce-lab"

    $ kubectl expose deployment/petclinic-tce-lab --port 80 --target-port 8080 --type=LoadBalancerkubectl expose deployment/petclinic-tce-lab --port 80 --target-port 8080 --type=LoadBalancer

  • List running Kubernetes Services, and you will see that in the Service named "petclinic-tce-lab" has an EXTERNAL-IP, which specifies the AWS elastic load balancer (ELB) endpoint that we can use to access your Pet Clinic application, like below:

    $ kubectl get svckubectl get svc

4.6 Access application from web browser

Copy the EXTERNAL-IP from the petclinic-tce-lab Service, and connect to it in your web browser of choice. It may take a minute or two to get the URL Routed, so if the webapp does not show up, just keep refreshing your browser.
