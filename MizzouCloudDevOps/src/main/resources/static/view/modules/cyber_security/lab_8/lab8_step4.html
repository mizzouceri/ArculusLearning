3.2 Monitoring System information with Node Exporter

  • Installing and running the Node Exporter

    Leave Prometheus and Grafana running and connect to EC2 instance through new console and execute below commands to install Node Exporter

    $ export node_exporter_ver=1.3.1
$ wget https://github.com/prometheus/node_exporter/releases/download/v${node_exporter_ver}/node_exporter-${node_exporter_ver}.linux-amd64.tar.gz
$ tar xvfz node_exporter-${node_exporter_ver}.linux-amd64.tar.gz
$ mv node_exporter-${node_exporter_ver}.linux-amd64 node_exporter

    Start Node Exporter: Navigate inside the installation directory of node exporter and start the node exporter using below commands:

    cd node_exporter
cd node_exporter
    ./node_exporter
./node_exporter

    You should see output like this indicating that the Node Exporter is now running and exposing metrics on port 9100:

  • Node Exporter metrics: Once the Node Exporter is installed and running, you can verify that metrics are being exported by cURLing the /metrics endpoint by connecting to the EC2 console once again

    curl http://localhost:9100/metrics
curl http://localhost:9100/metrics

    You should see output like this:

    Success! The Node Exporter is now exposing metrics that Prometheus can scrape, including a wide variety of system metrics further down in the output (prefixed with node_). To view those metrics (along with help and type information):

    curl http://localhost:9100/metrics | grep "node_"
curl http://localhost:9100/metrics | grep "node_"

  • Configuring your Prometheus Your locally running Prometheus instance needs to be properly configured in order to access Node Exporter metrics. The following prometheus.yml example configuration file will tell the Prometheus instance to scrape, and how frequently, from the Node Exporter via localhost:9100:

    Return back to the console in which prometheus is running and stop server using "Ctrl+C" command and open the prometheus.yml file using editor of your choice. Here we use nano editor as it is simple to use

    nano prometheus.yml
nano prometheus.yml

    Update the configuration(targets) with the following in the "scrape_configs" section:

    scrape_configs:
- job_name: node
    static_configs:
    - targets: ['localhost:9100']

    • After updating the port, save the changes using command "Ctrl+S" and exit the nano editor using command "Ctrl+X"

  • Start Prometheus server Start the prometheus server using the --config.file flag to point to the Prometheus configuration that you created above:

    ./prometheus --config.file=./prometheus.yml
./prometheus --config.file=./prometheus.yml

  • Exploring Node Exporter metrics

    Now that Prometheus is scraping metrics from a running Node Exporter instance, you can explore those metrics using the Prometheus UI (aka the expression browser). Navigate to http://VM_IP:9090/graph in your browser and use the main expression bar at the top of the page to enter expressions.

    Metrics specific to the Node Exporter are prefixed with node_ and include metrics like node_cpu_seconds_total and node_exporter_build_info.
