In this chapter, you will create the launch the Ansible nodes using AWS EC2 services, and then configure the nodes to speak to each other using Secure Shell (SSH) concepts.

2.1 AWS VM instance preparation

2.1.1 AWS EC2 overview

Figure 2.1 shows the instance architecture to be configured in this Lab. Using your AWS account, you will launch three EC2 instances that will be used to perform this lab. You will assign a key pair to be able to SSH into these instances from your computer. You will also create a security group to write different inbound and outbound rules. ; all the infrastructure will be created in a specific zone.

2.1.2 Create Ansible nodes on AWS

As mentioned earlier, you will be starting with creation of a key pair that will be used to gain a command line access to the EC2 instances. Then you will set up a security group with different rules. You will then go through the process of launching the AWS instances for this lab.

Access AWS EC2 Service

  • Login to your Amazon AWS cloud account,
  • Make sure to select the US East (N. Virginia) region in the top-right part of your screen. Then click "Services" on the top left corner and select "EC2" service (Elastic Compute Cloud) under the "Compute" or simply type 'EC2' to access the service (See Fig. 2.2).

Figure 2.2: Accessing to the EC2 Service.

Manage Key Pairs

  • In left menu, select "Key Pairs" under "Network & Security" option as shown in 2.3.

Figure 2.3: EC2 key pairs and security group navigation.

  • Click on "Create Key Pair" and create a key pair with a name you like, select "RSA", and "pem/ppk" basedonyour SSH client type ("pem" for Mac and "ppk" for Win10 with PuTTY, See Fig. 2.4). Additionally, the "pem" file can be used with MobaXterm in Win10. Once the key pair is created, download and store your .pem/.ppk file somewhere in your local computer, you will need this file to login to your EC2 instance and for Ansible nodes communication.

Figure 2.4: Creating Key Pair in EC2.

Manage Security Group

  • Select "Security Groups" as shown in 2.3, and click "create security group". Give it a name that you like, add any description, and choose the inbound rules for this security group as shown in Fig. 2.5 For this lab, you can set the inbound/outbound rules to allow all traffic/protocol/ports and set the Source IP range to 0.0.0.0/0.

Figure 2.5: Setting inbound rules in Security group

Launch Ansible Instances

For this lab, you will need three instances, one for ansible engine and two for ansible nodes. There is no difference in the configuration type and allocation settings, so you can initiate the launch of all the three instances at once.

  • From the left navigation menu in EC2, select 'Instances' to create instances and then click on 'Launch Instance' from the right-top.
  • Setting the tags is not required, but is a best practice. You can give module = ansible as the key pair if you want.

  • Choose the Amazon Linux 2 AMI (HVM) that you can see under 'Quick start' under the section 'Application and OS Images (Amazon Machine Image)'. Keep default for architecture

  • Under the 'Instance Type', select 't2.micro (Free Tier Eligible)'

  • Under the 'Key pair (login)', from the dropdown, click on the key you created before

  • In 'Network settings', click "Select an existing security group" and select the security group you just created;

  • In 'Configure storage', change the size to 8GiB of 'gp2' Root volume.

  • On the right hand side, in 'Summary', change the number of instances from 1 to 3. Make sure you have the right settings, and then click 'Launch instance'. Click on 'Launch Instance'. This will create the instances.

  • You will be shown a 'success' message, and click button to 'View all instance'.

  • In a short time your new instance will be deployed and ready for you to use. Make sure to change the names of the instances to ansible, node1 and node2. Note down the "Public IPv address" of the VM instance in the "Details". You will need the IP address to login to the VM instance.

2.2 Configuring Ansible Control Node

In this section, you will configure the Ansible Control node, and install Ansible as well as required libraries. Ansible Control instance is where you will be installing the Ansible framework to get started with Ansible.

2.2.1 Upload your key pair to the Ansible Control node

You will use SSH login for the Ansible engine and the nodes to work together in this lab. For this purpose, we will upload the private key downloaded from AWS to the engine instance, so it can login to the nodes in the topology without having to generate new key=pairs. To upload the key, do the following from your local machine where your private key is stored: 

$ scp -i "Your_Private_Key" "Your_Private_Key" ec2-user@"Your_Instance_Public_IP":~/.ssh/scp -i "Your_Private_Key" "Your_Private_Key" ec2-user@"Your_Instance_Public_IP":~/.ssh/

For example, for this case: 

$ scp -i ansibleKey ansibleKey ec2-user@34.200.214.26:~/.ssh/scp -i ansibleKey ansibleKey ec2-user@34.200.214.26:~/.ssh/

2.2.2 Login to the Ansible Control node

You will be using the key-pair that you created during AWS instance creation steps to login to the instance. Change the two quoted variables to your own values. 

$ ssh -i "Path/Your_Private_Key" ec2-user@"Your_Instance_Public_IP"ssh -i "Path/Your_Private_Key" ec2-user@"Your_Instance_Public_IP"

For example, for this case:

$ ssh -i ansibleKey ec2-user@34.200.214.26ssh -i ansibleKey ec2-user@34.200.214.26
