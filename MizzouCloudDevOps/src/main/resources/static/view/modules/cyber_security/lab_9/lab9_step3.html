2.1 Instance Preparation

Note: Certifier Framework is designed to run in Linux environments. This is primarily because of:
  • System calls and dependencies: It relies on Linux-specific system interfaces and libraries (like libssl, libprotobuf, etc.).
  • Build tools and scripts: The setup uses build tools like make, cmake, and dependencies typically installed via apt, which are native to Debian/Ubuntu or similar Linux distributions.
  • Enclave Support: Intel SGX enclaves require Linux for most development environments and SDK/tooling.

    • 2.1.1 Go to AWS and Sign In using your credentials.

    2.1.2 In AWS Console, select your region (e.g., N. Virginia).

    2.1.3 Search for EC2 in the services bar and open EC2 Dashboard.

    2.1.4 In EC2 Dashboard, click on AMIs in the left menu.

    2.1.5 Search: certifier_fw_image, filter by Public Images.

    2.1.6 Select and click Launch instance from AMI.

    2.2 Launch an Instance

    2.2.1 Create your virtual machine (EC2 instance):

    2.2.2 Enter a unique name (e.g., sample_instance) under "Name and tags".

    2.2.3 Ensure the AMI settings are preserved and visible under OS Image.

    2.2.4 Change the Instance Type to t2.medium.

    2.2.5 Create a new key pair and download it locally.

    Note: The created key pair will be downloaded locally.

    2.2.6 Leave default network and storage settings as inherited from AMI.

    2.2.7 Review instance summary on the left and click Launch Instance.

    2.2.8 After launching, click on Connect to your instance.
