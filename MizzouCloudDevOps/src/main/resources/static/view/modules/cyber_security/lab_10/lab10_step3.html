2.1 AWS EC2 Overview

Amazon EC2 (Elastic Compute Cloud) provides scalable virtual servers in the cloud. In this section, we will launch an EC2 instance using a pre-configured AMI image (that includes Jupyter, Python, and CUDA drivers), connect to the instance, and start a Jupyter Notebook server with GPU support.

2.2 Launch EC2 Instance from AMI

Access EC2 Service

  • Sign in to your AWS Management Console
  • Make sure to select the US East (N. Virginia) region in the top-right part of your screen.

  • In the top search bar, type EC2 and select EC2 from the services list.

  • In the left sidebar, select the AMI option under Images.

  • Select the AMI image by searching for AL_MCD or searching for AMI_ID: ami-090255533a40e2594, then select Launch Instance from AMI.

2.3 Configuring the Instance

  • To configure the instance, give it a name that you like. Make sure to select ml.g4dn.4xlarge as the instance type (GPU-supported).

  • Under Key pair (login), choose Create a new key pair if you don’t have one:
    • Give it any name of your choice
    • Key pair type: RSA
    • Private key format: .pem
    • Click Create key pair. This will download the .pem file to your computer.

Note: Save this .pem file in a secure location. You’ll need it to SSH into your instance.

All the other sections will remain the same. Once this is done, click Launch instance.

2.4 Adding Inbound Rules to Security Group

To access your EC2 instance from your local machine, you need to add inbound rules to the security group associated with your instance. These rules define which ports are open and who can access them.

We will add the following inbound rules:

  • Jupyter Notebook (port 8888): For running notebooks in your browser.
  • MLflow UI (port 5050): For accessing the MLflow experiment dashboard.

Once your instance is up, locate your instance from the Instance tab (left sidebar). Click on your instance to find the details. Select the Security tab and copy your security group number.

Now navigate to the left sidebar again and scroll down to Network & Security, then select Security Groups.

In the Find security groups by attribute or tag field, paste your copied security group code. Now, click the Inbound rules tab and select Edit inbound rules.

Add the following rules:

Type Protocol Port Range Source Description
Custom TCP TCP 8888 Custom: 0.0.0.0/0 For Jupyter Notebook
Custom TCP TCP 5050 Custom: 0.0.0.0/0 For MLflow UI

2.5 Connecting to Your Instance

Once your EC2 instance is launched and running, follow the steps below to connect to the instance securely, activate the GPU-enabled environment, and start a Jupyter Notebook server.

2.5.1 Set Permissions for Your PEM File (One-Time Step)

Before connecting, you must ensure that your .pem key has the correct file permissions. Open a terminal on your local machine and run the following command:

chmod 400 '/path/to/your.pem'

This command ensures your private key is not publicly viewable, which is required by SSH.

2.5.2 Connect to the EC2 Instance via SSH

Now, use the following SSH command to connect to your EC2 instance:

ssh -i '/path/to/your.pem' ubuntu@<YOUR-PUBLIC-IP>

Note: If this is your first time connecting, you may be prompted to confirm the authenticity of the host. Type yes and press Enter.

2.5.3 Activating the Conda Environment

Once you’re logged into the instance, activate the pre-installed Conda environment that supports GPU and deep learning frameworks:

conda activate torch-gpu

You should now be in the torch-gpu environment, which includes libraries like PyTorch with CUDA support.

To check if the environment setup is successful, run:

nvidia-smi

2.5.4 Launch Jupyter Notebook

To start a Jupyter Notebook server on the EC2 instance, use:

jupyter notebook --no-browser --port=8888 --ip=0.0.0.0

Important: Copy the token shown in the output. This token authenticates your session.

2.5.5 Access the Jupyter Notebook from Your Browser

  • Open your web browser.
  • Type the following URL into the address bar:
http://<YOUR-PUBLIC-IP>:8888
  • You should now see the Jupyter Notebook interface. Paste the token you copied earlier to access the interface.

2.6 Starting the MLflow UI

  • To access the MLflow UI, open a new terminal and SSH back into EC2:
ssh -i '/path/to/your.pem' ubuntu@<YOUR-PUBLIC-IP>
  • Once inside, activate the Conda environment again:
conda activate torch-gpu
  • Install a necessary package required by MLflow:
pip install "pydantic<2.0,>=1.10.13"
  • Run the following command to start the MLflow tracking server on port 5050:
mlflow ui --backend-store-uri file:///home/ubuntu/mlruns --host 0.0.0.0 --port 5050
  • Then, open your browser and go to:
http://<YOUR-PUBLIC-IP>:5050

You should now see the MLflow dashboard where you can monitor experiments, view metrics, and compare model runs.

Note: If you don’t see the dashboard, try running the following command in the terminal and reload the page:

rm -rf ~/mlruns
